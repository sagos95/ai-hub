# Session Retro — самообучение на основе сессии

Анализирует рабочую сессию, выявляет неоптимальные моменты и вносит улучшения в инструкции, скрипты и memory.

## Режимы работы

### Режим 1: Текущая сессия (без аргумента)

Если `$ARGUMENTS` пуст — анализируй текущую сессию. Весь контекст уже в окне.

### Режим 2: Завершённая сессия (с session_id)

Если `$ARGUMENTS` содержит session ID:

1. Найди файл сессии:
   ```bash
   ls ~/.claude/projects/$(echo $PWD | sed 's|/|-|g')/$ARGUMENTS.jsonl
   ```

2. Извлеки диалог (user + assistant сообщения):
   ```bash
   python3 -c "
   import json, sys
   messages = []
   for line in open(sys.argv[1]):
       obj = json.loads(line)
       if obj.get('type') in ('user', 'assistant'):
           role = obj['type']
           content = obj.get('message', {}).get('content', '')
           if isinstance(content, str) and content.strip():
               messages.append(f'## [{role.upper()}]\n{content[:3000]}')
           elif isinstance(content, list):
               texts = [b.get('text','') for b in content if b.get('type')=='text']
               if texts:
                   messages.append(f'## [{role.upper()}]\n{chr(10).join(texts)[:3000]}')
   print(chr(10).join(messages))
   " ~/.claude/projects/$(echo $PWD | sed 's|/|-|g')/$ARGUMENTS.jsonl
   ```

3. Если файл слишком большой — используй Agent tool для чтения и суммаризации.

## Фаза 1: Анализ

Проанализируй сессию по 5 категориям:

### 1. Потери времени
- Лишние шаги, повторные попытки, хождение по кругу
- Долгие поиски того, что можно было найти быстрее
- Запросы к пользователю, которые можно было избежать

### 2. Фидбек пользователя
- Моменты, где пользователь поправил агента (явно или неявно)
- Фразы типа «нет, не так», «лучше сделай...», «ты забыл...»
- Замечания по стилю, тону, подходу

### 3. Знания о проекте
- Новые факты о сервисах, архитектуре, процессах
- Связи между сервисами, которые не были очевидны
- Особенности рабочего процесса команды

### 4. Workflow-баги
- Шаги в инструкциях/скиллах, которые не сработали или были пропущены
- Недостающие шаги, которые пришлось добавлять по ходу
- Неточности в описаниях команд

### 5. Инструменты и скрипты
- Скрипты, которые упали или вернули неожиданный результат
- Недостающие скрипты/утилиты, которые бы ускорили работу
- MCP-инструменты, которые использовались неоптимально

## Фаза 4: Реализация

По согласованию с пользователем вноси изменения:

- **Memory** — создавай/обновляй файлы в `~/.claude/projects/$(echo $PWD | sed 's|/|-|g')/memory/`
- **Workflow/скиллы** — редактируй файлы в `integrations/*/commands/`
- **Скрипты** — редактируй/создавай в `integrations/*/scripts/`
- **CLAUDE.md** — обновляй корневой CLAUDE.md при необходимости
- **ORG-SERVICES-INDEX.md** — дополняй при обнаружении новых фактов о сервисах

## Важно

- Не дублируй то, что уже есть в memory или CLAUDE.md
- Приоритезируй: сначала исправления workflow-багов (они повторятся), потом знания, потом оптимизации
- Каждое изменение должно быть конкретным и actionable — не пиши абстрактные рекомендации
- При редактировании workflow-файлов проверяй, что симлинки в `.claude/commands/ai-hub/` ведут на правильные файлы
