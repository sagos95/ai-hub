# DAP Setup — бутстрап скиллов Dodo AI-платформы

Готовит машину к работе с **Dodo AI-платформой (DAP)**: ставит платформенные скиллы
`dap-doctor` (диагностика окружения и доступов) и `dap-shipmaster` (аллокация ресурсов →
манифест → CI/CD → вывод на прод) в `~/.claude/skills/`, затем приводит окружение в порядок
**строго по выводу `dap-doctor`**.

Эта команда — тонкая обёртка-инсталлятор поверх `dap-doctor`. Вся реальная диагностика и
команды-фиксы приходят из самого доктора (`check-env.sh`), а не захардкожены здесь — поэтому
флоу не устаревает вместе с платформой.

> **Это внешняя интеграция.** Скиллы качаются из GitHub-орг `dodo-ai-platform`
> (репо `project-template`). Нужен доступ к этой орг (выдаётся платформенной командой DAP)
> и `git`. Скиллы ставятся **глобально** в `~/.claude/skills/`, а не в этот репозиторий.

## Главное правило

**Действуй пошагово и спрашивай юзера перед КАЖДОЙ установкой или логином.** Ничего не ставь
и не логинься молча. На каждом `fix:` из доктора — сначала покажи команду юзеру и получи «да».

## Workflow

### Шаг 0: Предпосылки

Скажи юзеру кратко: «Сейчас поставлю скиллы Dodo AI-платформы и проверю окружение. Перед
каждой установкой/логином буду спрашивать. Нужен доступ к GitHub-орг **dodo-ai-platform** —
он должен быть уже выдан платформенной командой.»

Если юзер говорит, что доступа к орг ещё нет — остановись и попроси сначала получить доступ
(шаг вне этого флоу, делает платформенная команда DAP).

### Шаг 1: git

`git` нужен, чтобы забутстрапить доктора.

```bash
command -v git && git --version
```

- Если `git` есть — переходи к Шагу 2.
- Если нет **на macOS** — git входит в Command Line Tools, **не** Homebrew. Спроси согласие
  и предложи юзеру выполнить:
  ```bash
  xcode-select --install
  ```
  Это откроет GUI-установщик. Дождись, пока юзер скажет «готово», и проверь `git --version` снова.
- Если нет **не на macOS** — НЕ ставь сам. Покажи юзеру команду для его ОС
  (`apt install git` / `dnf install git` / и т.п.) и попроси установить вручную.

### Шаг 2: Бутстрап скиллов платформы

Нужен только `git` + доступ к орг DAP. Спроси согласие, затем выполни:

```bash
git clone --depth 1 https://github.com/dodo-ai-platform/project-template /tmp/dap-template \
  && bash /tmp/dap-template/.agents/skills/dap-doctor/scripts/install-skills.sh
```

Это разложит в `~/.claude/skills/` две скиллы: **dap-doctor** и **dap-shipmaster**.

Разруливание ошибок (переводи юзеру по-человечески):
- `Permission denied` / `Repository not found` при clone → нет доступа к орг `dodo-ai-platform`.
  Скажи юзеру попросить доступ у платформенной команды DAP и вернуться.
- `/tmp/dap-template already exists` → `rm -rf /tmp/dap-template` и повтори clone.
- Скрипт `install-skills.sh` не найден → проверь, что репо склонировался
  (`ls /tmp/dap-template/.agents/skills/dap-doctor/scripts/`).

После успеха убедись, что доктор на месте:
```bash
test -f ~/.claude/skills/dap-doctor/scripts/check-env.sh && echo "dap-doctor: ok"
```

### Шаг 3: Tier-1 диагностика и фиксы

Запусти диагностику доктора:

```bash
bash ~/.claude/skills/dap-doctor/scripts/check-env.sh
```

Скрипт печатает строки `[ ok ]` / `[WARN] ` / `[MISS]`, и у проблемных есть готовая команда
после `fix:`. Проверяются `docker` / `git` / `gh` / `kubectl` / `kubelogin` и то, что доступы
реально работают (gh auth, kube API через OIDC-логин в браузере, GHCR).

**Цикл починки** (по одному, с согласия юзера):

1. Для каждой строки `[MISS]` / `[WARN]` возьми её `fix:`-команду.
2. Покажи команду юзеру и спроси согласие **перед запуском** (особенно для установок и логинов —
   `gh auth login`, OIDC-логин в браузере и т.п.).
3. Выполни ровно этот `fix:` — не импровизируй, не объединяй несколько фиксов в один.
4. Перезапусти `check-env.sh` и продолжай, **пока всё не станет зелёным** (`[ ok ]`).

**Важно про ОС:** на **не-macOS** ничего не ставь сам — просто покажи юзеру `fix:`-команду
и попроси выполнить вручную, затем перезапусти диагностику.

Логины (gh / kube OIDC / GHCR) — это всегда отдельный явный шаг с подтверждением юзера.
Не дергай токены и не логинься в фоне.

### Шаг 4: Поддержание скиллов в актуальном состоянии

Скажи юзеру, что обновлять платформенные скиллы нужно тем же доктором — периодически
(раз в неделю или после анонсов платформенной команды):

```bash
# сравнить версии с эталоном
bash ~/.claude/skills/dap-doctor/scripts/install-skills.sh --check
# установить/обновить
bash ~/.claude/skills/dap-doctor/scripts/install-skills.sh
```

Если юзер просит обновить прямо сейчас — сначала `--check`, покажи разницу, спроси согласие,
потом обнови.

## Итог

Когда `check-env.sh` зелёный по всем строкам — окружение готово. Скажи юзеру:
«DAP готов. Скиллы `dap-doctor` и `dap-shipmaster` установлены в `~/.claude/skills/`.
Чтобы они подхватились, **перезапусти сессию агента** (MCP/скиллы грузятся при старте).
После рестарта можно запускать `dap-shipmaster` для вывода сервиса на прод.»
