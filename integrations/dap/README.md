# DAP — Dodo AI-Platform bootstrap

Интеграция-инсталлятор для **Dodo AI-платформы (DAP)**. Ставит платформенные скиллы и
приводит окружение машины в порядок, опираясь на платформенный скилл `dap-doctor`.

В отличие от остальных интеграций AI Hub (Kaiten, Time, Buildin — источники данных), `dap` —
это **бутстрап внешних скиллов**: он не читает данные из DAP, а раскладывает в `~/.claude/skills/`
две скиллы платформы и чинит окружение по их диагностике.

## Что ставит

| Скилл платформы | Назначение |
|-----------------|------------|
| **dap-doctor** | Диагностика окружения и доступов (`check-env.sh`), установка/обновление скиллов (`install-skills.sh`) |
| **dap-shipmaster** | Аллокация ресурсов → манифест → CI/CD → вывод на прод |

Скиллы качаются из GitHub-орг **`dodo-ai-platform`** (репо `project-template`) и ставятся
**глобально** в `~/.claude/skills/`, а не в этот репозиторий.

## Использование

```
/ai-hub:dap-setup
```

или просто попроси агента: «настрой Dodo AI-платформу», «поставь скиллы платформы»,
«забутстрапь доктора».

Также `/ai-hub:setup` предлагает установку DAP опциональным шагом — если согласишься,
запустится тот же флоу.

## Требования

- `git` (на macOS — из Command Line Tools: `xcode-select --install`, **не** Homebrew).
- Доступ к GitHub-орг `dodo-ai-platform` (выдаёт платформенная команда DAP).
- Окружение, которое проверяет доктор: `docker`, `gh`, `kubectl`, `kubelogin` + рабочие
  доступы (gh auth, kube API через OIDC, GHCR). Всё, чего не хватает, доктор подскажет
  командой `fix:`.

## Флоу

1. **git** — проверить/поставить.
2. **Бутстрап** — `git clone` шаблона + `install-skills.sh` → две скиллы в `~/.claude/skills/`.
3. **Tier-1 диагностика** — `check-env.sh`, чинить каждый `[MISS]`/`[WARN]` его `fix:`-командой
   (по одному, с согласия юзера), пока всё не зелёное. На не-macOS — не ставить самому.
4. **Обновление** — периодически `install-skills.sh --check` / `install-skills.sh`.

Подробный пошаговый флоу — в [`commands/dap-setup.md`](commands/dap-setup.md).

## Безопасность

- Любая установка и любой логин (`gh auth login`, kube OIDC, GHCR) — только с явного
  согласия юзера, по одному шагу.
- На не-macOS интеграция ничего не ставит сама — только показывает команду.
